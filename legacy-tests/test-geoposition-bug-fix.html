This test demonstrates the fix for the error:

TypeError: GeocodingState: position must be a GeoPosition instance or null

Issue Location

  • File: src/coordination/WebGeocodingManager.js
  • Line: 709 (before fix)
  • Method: getSingleLocationUpdate()

Root Cause

GeolocationService returns a raw browser position object, but GeocodingState.setPosition() expects a GeoPosition instance.

The Fix

Wrap the raw browser position in a GeoPosition instance before passing to GeocodingState:

const geoPosition = new GeoPosition(position);
