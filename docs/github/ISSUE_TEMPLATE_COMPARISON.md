# Issue Template Comparison

Overview of the issue templates available in this repository and when to use each.

## Available Templates

| Template | File | Use when |
|----------|------|----------|
| Bug report | `bug_report.md` | A feature is broken or behaves unexpectedly |
| Feature request | `feature_request.md` | You want to propose new functionality |
| Functional specification | `functional_specification.md` | Specifying detailed behaviour before implementation |
| GitHub config | `github_config.md` | Configuring GitHub-specific settings or workflows |

## Choosing the Right Template

- **Bug**: You observed unexpected behaviour and can reproduce it.
- **Feature**: You have an idea for an improvement; no bug involved.
- **Functional spec**: You are defining acceptance criteria for a planned change.
- **GitHub config**: You need to update CI/CD, branch protection, or other GitHub settings.
