🎤 Teste: Fala do Endereço Completo a Cada 50 Segundos

✅ Feature Implementada!

O aplicativo agora fala o endereço completo a cada 50 segundos, conforme solicitado no issue.

📋 Descrição da Implementação

Como Funciona:
  1. O PositionManager monitora a localização do usuário
  2. Quando passam ≥50 segundos desde a última atualização, envia evento strCurrPosUpdate
  3. O ReverseGeocoder recebe esse evento e faz a geocodificação reversa
  4. O HtmlSpeechSynthesisDisplayer detecta strCurrPosUpdate
  5. O sistema fala o endereço completo usando SpeechSynthesis API
⚠️ Nota Importante:

Este teste é conceitual pois requer:

  • Permissão de geolocalização do navegador
  • Movimento real do usuário (mudança de coordenadas)
  • API SpeechSynthesis suportada pelo navegador
  • Conexão com a API OpenStreetMap/Nominatim

🎯 Casos de Uso

Cenário: Usuário Dirigindo pela Cidade

  1. T=0s: Usuário inicia o aplicativo na Rua Augusta
  2. T=50s: 🔊 "Você está em: Rua Augusta, número 1200, Bela Vista, São Paulo"
  3. T=60s: Usuário vira para a Rua Oscar Freire
  4. T=61s: 🔊 "Você está agora em Rua Oscar Freire" (mudança de logradouro - prioridade 1)
  5. T=110s: 🔊 "Você está em: Rua Oscar Freire, Jardim Paulista, São Paulo" (50s após última fala completa)

Prioridades de Fala

  • Prioridade 3: Mudança de município (ex: São Paulo → Rio de Janeiro)
  • Prioridade 2: Mudança de bairro (ex: Bela Vista → Jardim Paulista)
  • Prioridade 1: Mudança de logradouro (ex: Rua Augusta → Rua Oscar Freire)
  • Prioridade 0: Endereço completo a cada 50 segundos ✨ (novo!)
