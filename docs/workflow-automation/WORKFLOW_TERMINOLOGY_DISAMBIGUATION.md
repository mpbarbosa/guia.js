The guia_js repository uses the term **"workflow"** in **two completely different contexts**, which can cause confusion:

1. **`.ai_workflow/`** - AI automation logs and artifacts (external tool)

#### Workflow Purposes

##### 1. `copilot-coding-agent.yml` (Primary Validation)
**Triggers**: Push to main/develop, PRs to main  
**Duration**: ~30-60 seconds  
**Jobs**:
- JavaScript syntax validation (`node -c`)
- Basic functionality test (`node src/guia.js`)
- Web server connectivity test
- Required files verification
- Code style checks
- Security scanning

**Example**:
```yaml

  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

  validate:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
      
      - name: Setup Node.js

          node-version: '18'
      
      - name: Validate JavaScript syntax
        run: node -c src/guia.js && node -c src/guia_ibge.js
```

##### 2. `documentation-lint.yml` (Documentation Quality)
**Triggers**: .md file changes, PRs, push to main  
**Duration**: ~1-3 minutes  
**Jobs**:
- Markdown linting (markdownlint-cli2)
- Link validation (internal/external)
- Line number deprecation checks
- Badge synchronization
- Version consistency validation

**Example**:
```yaml

  pull_request:

      - '**.md'

    branches:

          globs: '**/*.md'
```

##### 3. `modified-files.yml` (Smart Change Detection)
**Triggers**: Push/PR to main/develop  
**Duration**: ~2-5 minutes  
**Jobs**:
- Change detection (JS, tests, docs, src)
- Affected tests execution
- Test documentation updates
- Documentation validation
- Coverage badge updates
- Workflow summary generation

**Example**:
```yaml
name: Modified Files - Test and Documentation Updates

on:
  pull_request:
    branches: [ main, develop ]
  push:
    branches: [ main, develop ]

jobs:
  detect-changes:
    name: Detect File Changes
    runs-on: ubuntu-latest
    outputs:
      js_files: ${{ steps.filter.outputs.js }}
      test_files: ${{ steps.filter.outputs.tests }}
    steps:
      - name: Detect changed files
        run: |
          git diff --name-only HEAD~1 HEAD
```

#### Characteristics
- **Executable YAML**: GitHub Actions interprets and runs these
- **Triggered Automatically**: On push, PR, schedule, or manual dispatch
- **Cloud Execution**: Runs on GitHub's infrastructure (ubuntu-latest)
- **Version Controlled**: Changes to workflows are tracked in git
- **Developer Modified**: Team actively edits and improves workflows

#### Purpose in Development Lifecycle
GitHub workflows:
1. **Validate** code changes before merge
2. **Run** automated tests on every commit
3. **Check** documentation quality
4. **Update** metadata and badges
5. **Prevent** broken code from reaching main branch
6. **Report** test results and coverage

**Key Point**: This is about **CI/CD automation**, not AI-assisted development.

## ğŸ”„ Workflow Interactions

### Do They Interact?
**Short Answer**: No

### Detailed Explanation

**AI Workflow** (`.ai_workflow/`):

- Analyzes repository state

- Triggered by git events
- Validates code changes
- Runs tests automatically
- **Does not read `.ai_workflow/` logs**
- **Does not trigger AI workflow tool**
